# Talking Skeleton

An AI-powered escape room prop. A skeleton in a coffin listens to players, understands them, and answers back - in Slovak - using its own voice.

Built with OpenAI Whisper, GPT-4o, and ElevenLabs. Runs on a Raspberry Pi 4 inside Docker, or directly on macOS for development.

---

## How it works

1. The skeleton passively listens via a USB microphone (connected through a USB audio card).
2. When a player speaks, it records until 1 second of silence.
3. The audio is transcribed to Slovak text via **Whisper**.
4. A short, in-character response is generated by **GPT-4o** using the skeleton's personality.
5. The response is synthesised to speech by **ElevenLabs** and streamed directly to the speakers.
6. If no one speaks for 20-40 seconds, a random **invitation clip** plays to prompt players to interact.

## Setup

### 1. Clone and enter the project

```bash
cd Skeleton
```

### 2. Create your `.env` file

```bash
cp .env.example .env
```

Fill in the values:

```env
OPENAI_API_KEY=sk-...
ELEVENLABS_API_KEY=...
ELEVENLABS_VOICE_ID=...
```

- **OpenAI key**: [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
- **ElevenLabs key**: [elevenlabs.io/app/settings/api-keys](https://elevenlabs.io/app/settings/api-keys)
- **Voice ID**: [elevenlabs.io/voice-library](https://elevenlabs.io/voice-library) - pick an old, morbid voice and copy its ID from the URL

### 3. Add invitation clips

Drop any audio files (MP3, WAV, OGG, FLAC) into the `invitation/` folder. These play randomly when no one has spoken for 20-40 seconds.

## Running locally (macOS)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

On first launch you will be prompted to select your audio input and output devices. The choice is saved to `.env` and shown again on every startup.

---

## Running on Raspberry Pi (Docker)

```bash
docker compose up --build -d
```

Audio is passed through to the container via ALSA (`/dev/snd`). Logs and fallback audio are persisted via Docker volumes.

## Configuration

All tuning lives in `config.py`:

| Setting                | Default | Description                                |
|------------------------|-------- |--------------------------------------------|
| `SILENCE_DURATION_SEC` | `1.0`   | Seconds of silence before recording stops  |
| `MAX_RECORDING_SEC`    | `10.0`  | Hard recording cutoff                      |
| `VAD_AGGRESSIVENESS`   | `3`     | 0-3, higher filters more background noise  |
| `IDLE_MIN_SEC`         | `20.0`  | Minimum silence before an invitation plays |
| `IDLE_MAX_SEC`         | `40.0`  | Maximum silence before an invitation plays |
| `VOICE_STABILITY`      | `0.50`  | Lower = more expressive voice              |
| `VOICE_SIMILARITY`     | `0.70`  | Higher = closer to the original voice      |
| `VOICE_STYLE`          | `0.20`  | Style exaggeration (0 = none, 1 = max)     |
| `VOICE_SPEED`          | `1.00`  | Speech speed (lower = slower)              |

## Personality

The skeleton's backstory and character are defined in `personality.md`. Edit this file to change who the skeleton is, what it knows, and how it speaks. GPT-4o uses it as the system prompt.

The current character is **Belle Gunness** - a 19th-century Norwegian-American serial killer.

To apply changes, just restart the script (no rebuild needed when running locally).
